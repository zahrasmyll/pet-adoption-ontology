Pet Adoption Ontology: SPARQL Query Library back to ToC

To evaluate the competency and practical utility of the Pet Adoption Ontology, a comprehensive set of SPARQL queries has been designed. These queries address specific Competency Questions (CQs) related to shelter management, health status tracking, budgeting constraints, and animal classification.

Query 1: List all pets and their assigned shelters (Relates to CQ1 & CQ2)

Retrieves all registered pets alongside the specific shelter where they are currently located.

PREFIX : <http://www.example.org/petadoption#>
SELECT ?petName ?shelterName
WHERE {
  ?pet rdf:type :Pet ;
       :petName ?petName ;
       :livesIn ?shelter .
  ?shelter rdfs:label ?shelterName .
}

Query 2: Identify adopters and their specific budget categories (Relates to CQ3)

Extracts the profile names of prospective adopters and maps them directly to their defined financial budget limits.

PREFIX : <http://www.example.org/petadoption#>
SELECT ?adopterName ?budgetType
WHERE {
  ?adopter rdf:type :Adopter ;
           :petName ?adopterName ;
           :canAfford ?budget .
  ?budget rdf:type :AdopterBudget .
}

Query 3: Retrieve all dogs using class hierarchy (Relates to CQ5)

Leverages TBox taxonomic reasoning to successfully list all instances explicitly declared as or inferred to be Dogs.

PREFIX : <http://www.example.org/petadoption#>
SELECT ?dogName
WHERE {
  ?dog rdf:type :Dog ;
       :petName ?dogName .
}

Query 4: Find pets suitable for Apartment living (Relates to CQ5)

Filters animal profiles based on their spatial environmental requirements, isolating pets suitable for "Small" living spaces.

PREFIX : <http://www.example.org/petadoption#>
SELECT ?petName
WHERE {
  ?pet :requiresEnvironment ?env .
  ?env rdf:type :LivingSpace ;
       :requiredSpaceSize "Small" .
  ?pet :petName ?petName .
}

Query 5: Match Adopters to Pets based on Budget (Relates to CQ3 & CQ8)

Performs semantic matching between low-budget adopters and pets whose estimated monthly maintenance costs do not exceed $100.

PREFIX : <http://www.example.org/petadoption#>
SELECT ?adopterName ?petName
WHERE {
  ?adopter :canAfford :LowBudget ;
           :petName ?adopterName .
  ?pet :hasMaintenanceCost ?cost .
  ?cost :estimatedMonthlyCost ?amount .
  FILTER(?amount <= 100)
  ?pet :petName ?petName .
}

Query 6: Count the number of pets in each shelter (Relates to CQ1)

Uses SPARQL aggregation functions to dynamically calculate the overall animal population density across individual animal shelters.

PREFIX : <http://www.example.org/petadoption#>
SELECT ?shelterName (COUNT(?pet) AS ?petCount)
WHERE {
  ?pet :livesIn ?shelter .
  ?shelter rdfs:label ?shelterName .
}
GROUP BY ?shelterName

Query 7: Average maintenance cost by Pet Category (Relates to CQ8)

Calculates and group-compares the mathematical mean of monthly care expenses specifically for the Dog and Cat sub-classes.

PREFIX : <http://www.example.org/petadoption#>
SELECT ?type (AVG(?amount) AS ?avgCost)
WHERE {
  ?pet rdf:type ?type .
  ?pet :hasMaintenanceCost ?cost .
  ?cost :estimatedMonthlyCost ?amount .
  FILTER(?type IN (:Dog, :Cat))
}
GROUP BY ?type

Query 8: Find shelters with the most vaccinated pets (Relates to CQ4 & CQ6)

Tracks operational medical statuses by listing shelters sorted by their volume of fully vaccinated animal residents.

PREFIX : <http://www.example.org/petadoption#>
SELECT ?shelterName (COUNT(?pet) AS ?vaccinatedCount)
WHERE {
  ?pet :livesIn ?shelter ;
       :hasHealthStatus ?health .
  ?health :isVaccinated "true"^^xsd:boolean .
  ?shelter rdfs:label ?shelterName .
}
GROUP BY ?shelterName
ORDER BY DESC(?vaccinatedCount)
